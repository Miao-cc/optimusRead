Sensitivity Test

Usage

  • simulateSimplePsr_mcc:

    • function:
      • (usage) simulate the pulsar with the parameters in the parameter file (called pulsar1.params)
    • usage:
      • ./simulatesimplepsr_mcc -p [observing sys parameter file .params] -p [pulsar parameter file .params] -o [binary file *.dat]
      • ./simulatesimplepsr_mcc -p FAST.params -p pulsar.params -o pulsar.dat
    • out: *.dat

  • beamWeightMould.py

    • function:
      • Return the beam weight function for FAST (useful for pulsar drifting scan data simulation).
    • usage1:
      • Out put a binary file noBeamWeight.dat. Edit the file to change out file name and parameter settings
      • python beamWeightMould.py
    • usage2:
      • Import the module in a python file. Return a numpy array (N x M, N:Spectra per file; M:Number of channels). Default input scale = 1, percent =0, flag = 1.
      • from beamWeightMould import *
#scale: Max value of the weight func (Normalized). (0 ~ 1, default: 1)
#percent: The time of center Drift func. (0 ~ 1, default: 0)
#flag: The func switch. 1: no using the drifting func. 0: using the drifting. (0/1, default: 1)

 >> from beamWeightMould import *
 >> beamWeight(filename, scale=1, percent=0, flag=1)

  • fitsio_combinePolBinary.py:

    • function:
      • combine fits file with binary file to a new file with one pol.
    • usage:
      • python *.py fitsfile binaryfile combinedfile

  • fitsio_combinePolBinaryV1.py

    • function:
      • combine fits file with binary file to a new file with one pol. Adding the beam weight moudle to the simulsation data.
    • usage1:
      • combine the fits file and binary file without beam weight.
      • python *.py fitsfile binaryfile combinedfile
    • usage2:
      • combine the fits file and binary file with beam weight.
      • python *.py fitsfile binaryfile beamWeightfile combinedfile

Prepare the files

  • fits file
  • paramter files
    • FAST.params
    • pulsar.params

FAST.params

name: FAST system
f1: 0.125
f2: 1024.125
nchan: 4096
t0: 0
t1: 52.4288
gain: 1
tsys: 30
raj: 0
decj: 0
useAngle: 0
tsamp: 0.0002

pulsar.params

name: J1713+0747
p0: 0.8
dm: 20
raj: 4.510914803
decj: 0.13602659 
width: 0.01
flux: 0.01
useAngle: 0

Simulate a pulsar

simulatesimplepsr_mcc -p FAST.params -p pulsar.params -o pulsar.dat

This will out put a binary file.

Create beam weight module

python beamWeightMould.py

#scale: Max value of the weight func (Normalized). (0 ~ 1, default: 1)
#percent: The time of center Drift func. (0 ~ 1, default: 0)
#flag: The func switch. 1: no using the drifting func. 0: using the drifting. (0/1, default: 1)

 >> from beamWeightMould import *
 >> beamWeight(filename, scale=1, percent=0, flag=1)

This python code can create a beam weight module file. The module file will be used in the python program fitsio_combinePolBinaryV1.py. For a drifting scan, changing of percent is necessary.

Adding a simple pulsar

With beam weight:

Combine fits file with binary file to a new file with one pol. Adding the beam weight moudle to the simulsation data.

python fitsio_combinePolBinaryV1.py fitsfile binaryfile beamWeightfile combinedfile

This will combine the fits file and binary file with beam weight. Combine logic:

  1. Open the noise file (fitsfile)
  2. Open the binary file created by simulatesimplepsr_mcc (binaryfile)
  3. Open the beam weight file created by beamWeightMould.py (beamWeightfile)
  4. combine the files to the fits file (combinedfile )

No beam weight:

python fitsio_combinePolBinary.py fitsfile binaryfile combinedfile

Fold data

For FAST UWB data

  1. cut file

    • Downsamp_FASTpsrfits_freq_time_splitpol.py (python cut files)
    • cut_downsamping.sh (shell cut files)
    • filelist.txt

  2. fold file

    • cutfilelist.txt
    • fold.sh (shell fold files)

  3. usage

    1. ls *.fits > filelist.txt
    2. sh cut_downsamping.sh
    3. ls pol1.fits > cutfilelist.txt
    4. add the period (sec) and dm to the cutfilelist.txt, example: test_DM_16.2_Flux_0.269_P0_4.0s_pol1_M.fits 4.0 16.2
    5. sh fold.sh

For FAST 19-Beam data

  1. fold file

    • cutfilelist.txt
    • fold.sh (shell fold files)

  2. usage

    1. ls *.fits > cutfilelist.txt
    2. add the period (sec) and dm to the cutfilelist.txt, example: test_DM_16.2_Flux_0.269_P0_4.0s_pol1_M.fits 4.0 16.2
    3. sh fold.sh
