Project layout

  • simulateCal：

  • simulateRFI：

  • simulateSimplePsr：

    • function:
      • (usage1) simulate the pulsar with the defaultparameters.this produces a binary data file (pulsar.dat)
      • (usage2) simulate the pulsar with the parameters in the parameter file (called pulsar1.params)
    • usage1:
      • ./simulatesimplepsr -o [binary file *.dat]
      • ./simulatesimplepsr -o pulsar.dat
    • usage2:
      • ./simulatesimplepsr -p [observing sys parameter file .params] -p [pulsar parameter file .params] -o [binary file *.dat]
      • ./simulatesimplepsr -p parkes1.params -p pulsar2.params -o pulsar2.dat
    • out: *.dat

  • simulateComplexpsr：

  • simulateSystemNoise：

    • function:
      • (usage1) will use default options for the properties of the noise (in this case the default is to produce Gaussian random noise with a mean of 0 and a standard deviation of 1).
      • (usage2) This will write, in plain text format, to the file snoise.txt.
      • (usage3) simulate system noise using input parameters in the *.params files
    • usage1:
      • ./simulateSystemNoise -o [binary file *.dat]
      • ./simulateSystemNoise -o snoise.dat
    • usage2: ./simulateSystemNoise -o [binary file .dat] -to [ASIC file .txt] [t0] [t1] [chan0] [chan1]
      • [t0]: [start fo time]; [t1]: [end fo time]; ===> time range between 0 and 3 seconds
      • [chan0]: [start num fo chnnels]; [chan1]: [end um fo chnnels]; ===> frequency channels between 0 and 3
      • ./simulateSystemNoise -o snoise.dat -to snoise.txt 0 3 0 3
    • usage3:
      • ./simulateSystemNoise -o [binary file .dat] -p [parameter file .params]
      • ./simulateSystemNoise -o snoise.dat -p parkes1.params
    • out: .txt ; .dat

  • createSearchFile：

    • function:
      • will load in the binary data file (snoise.dat) and produce a PSRFITS search mode file with filename "search.sf".
    • usage:
      • ./createSearchFile -o [outfile .sf] -f [binary file .dat]
      • ./createSearchFile -o search.sf -f snoise.dat
      • ./createSearchFile -o [outfile .sf] -p [parameter file .params] -f [binary file *.dat]
      • ./createSearchFile -o search.sf -p parkes1.params -p digitiser.params -f snoise.dat -f pulsar1.dat -f pulsar2.dat
    • out: *.sf

  • inspectBinaryFile：

    • function:
      • view binary files
    • usage:
      • ./inspectBinaryFile -f [binary file *.dat]
      • ./inspectBinaryFile -f snoise.dat
    • out:
      • will list the time span of the data file, the sampling time, the observing band, number of channels and so on.

  • simulateSimplePsr_mcc:

    • function:
      • (usage) simulate the pulsar with the parameters in the parameter file (called pulsar1.params)
    • usage:
      • ./simulatesimplepsr_mcc -p [observing sys parameter file .params] -p [pulsar parameter file .params] -o [binary file *.dat]
      • ./simulatesimplepsr_mcc -p FAST.params -p pulsar.params -o pulsar.dat
    • out: *.dat

  • beamWeightMould.py

    • function:
      • Return the beam weight function for FAST (useful for pulsar drifting scan data simulation).
    • usage1:
      • Out put a binary file noBeamWeight.dat. Edit the file to change out file name and parameter settings
      • python beamWeightMould.py
    • usage2:
      • Import the module in a python file. Return a numpy array (N x M, N:Spectra per file; M:Number of channels). Default input scale = 1, percent =0, flag = 1.
      • from beamWeightMould import *
#scale: Max value of the weight func (Normalized). (0 ~ 1, default: 1)
#percent: The time of center Drift func. (0 ~ 1, default: 0)
#flag: The func switch. 1: no using the drifting func. 0: using the drifting. (0/1, default: 1)

 >> from beamWeightMould import *
 >> beamWeight(filename, scale=1, percent=0, flag=1)

  • fitsio_combinePolBinary.py:

    • function:
      • combine fits file with binary file to a new file with one pol.
    • usage:
      • python *.py fitsfile binaryfile combinedfile

  • fitsio_combinePolBinaryV1.py

    • function:
      • combine fits file with binary file to a new file with one pol. Adding the beam weight moudle to the simulsation data.
    • usage1:
      • combine the fits file and binary file without beam weight.
      • python *.py fitsfile binaryfile combinedfile
    • usage2:
      • combine the fits file and binary file with beam weight.
      • python *.py fitsfile binaryfile beamWeightfile combinedfile

  • fitsio_cutfreq.py:

    • function:
      • cut file only have one pol.
    • usage:
      • python *.py filename startfref endfref newfile

  • fitsio_cut_freq_time_splitpol.py:

    • function:
      • cut file have two pols.
    • usage:
      • python *.py filename startchan endchan startTime endTime filename

  • fastFitsCut.py:

    • function:
      • cut file have two pols. Same function as fitsio_cut_freq_time_splitpol.py but more faster.

    • usage:

      • python fastFitsCut.py [-f1 startchannel] [-f2 endchannel] [-t1 startsubint] [-t2 endsubint] FAST.fits
      • python fastFitsCut.py -[help/version] example:
      • python fastFitsCut.py -f1 1000 -f2 3000 -t1 0 -t2 31 FAST.fits
      • python fastFitsCut.py -f1 1000 FAST.fits (From channel 1000 to end. All subint)

      (if someone lost, the input will be set to a default num.)

  • get_bandpass.py:

    • function:
      • get bandpass from fits file.
    • usage:
      • def function can be found in fitsio_combinePolBinary.py

  • get_nos.py:

    • function:
      • get signal and noise from fits file.
    • usage:
      • def function can be found in fitsio_combinePolBinary.py

  • plotbinary.py:

    • function:
      • plot one channel of the binary file(create by simulateSimplePsr_mcc).
    • usage:
      • python plotbinary.py binaryfile

  • plotfits.py:

    • function:
      • plot fast fits files.
    • usage:
      • python plotfits.py filename

  • plotfits2png.py

    • function:
      • plot fast fits files to png.
    • usage:
      • python plotfits2png.py filename
